Về SBOM (Software Bill of Materials)
SBOM là danh sách đầy đủ tất cả các thành phần phần mềm (libraries, frameworks, packages) trong ứng dụng. Được dùng để nhanh chóng xác định tác động khi phát hiện lỗ hổng mới (ví dụ: Log4Shell, XZ Utils). Format CycloneDX 1.6 tương thích với GitLab, NIST, và các SBOM consumers khác.

📄 Dockerfile Hardening — Before vs After

❌ Dockerfile gốc (KHÔNG AN TOÀN)
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 5000
CMD ["npm", "run", "dev"]

# ⚠️ Vấn đề:
# - Root user (privilege escalation risk)
# - npm install (thay vì npm ci)
# - Node 18 (EOL, nhiều CVEs hơn)
# - Không có healthcheck
# - Single-stage (dev tools trong prod)
# - Không có security labels
✅ Dockerfile hardened (AN TOÀN)
# Multi-stage build
FROM node:20-alpine AS builder
RUN apk add --no-cache dumb-init
WORKDIR /app
COPY package*.json ./
RUN npm ci --ignore-scripts
COPY . .
RUN rm -f .env .env.*

FROM node:20-alpine AS production
LABEL security.hardened="true"
RUN npm ci --only=production
RUN adduser -S nodeuser -u 1001
USER nodeuser
EXPOSE 4000
HEALTHCHECK --interval=30s CMD node...
ENTRYPOINT ["dumb-init", "--"]
CMD ["node", "server.js"]
